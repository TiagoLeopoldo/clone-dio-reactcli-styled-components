# Projeto Clone DIO - Plataforma de Estudos (React.JS Vite - Styled-components)

Este projeto foi desenvolvido como parte de um bootcamp da **Digital Innovation One (DIO)**, simulando uma plataforma de estudos com autenticação, feed de posts e ranking de usuários.

[**(Deploy - Vercel)**](https://clone-dio-reactcli-styled-component.vercel.app/)

### **Arquitetura Aplicada**

- **Vite** com **React 19**
- **Styled-components** para estilização
- **React Hook Form** + **Yup** para formulários
- **Axios** para consumo de APIs
- Organização baseada em componentes (não estritamente Atomic Design)

### **Início Rápido**

```bash
git clone https://github.com/TiagoLeopoldo/nome-do-repositorio.git
cd nome-do-repositorio
yarn install
yarn dev       # Frontend
yarn api       # Backend (JSON-Server)
```

Acesse: [http://localhost:5173](http://localhost:5173)

### **Por que Vite? (Vite vs React CLI)**

O projeto foi inicialmente pensado para utilizar **React CLI**, mas optou-se por **Vite** devido a:

- **Performance superior:** inicialização e hot reload muito mais rápidos
- **Produtividade:** menos tempo de espera para testes visuais e validações (isso me economizou horas de espera desnecessária para renderizar a página)
- **Experiência do desenvolvedor:** fluxo de trabalho mais ágil sem comprometer compatibilidade com React

### **Tecnologias Utilizadas**

| Categoria   | Tecnologias          |
| ----------- | -------------------- |
| Core        | React 19, Vite       |
| Estilização | Styled-components    |
| Formulários | React Hook Form, Yup |
| HTTP        | Axios                |
| Roteamento  | React Router DOM     |
| Ícones      | React Icons          |
| API         | JSON-Server          |

### **Estrutura do Projeto**

```
src/
├── components/    # Componentes reutilizáveis
├── pages/         # Páginas da aplicação
├── services/      # APIs e serviços
├── styles/        # Estilos globais
└── ...
```

### **Páginas**

- **Home** - Página inicial com chamada para ação
- **Login** - Autenticação de usuários
- **SignUp** - Cadastro de novos usuários
- **Feed** - Posts e ranking de usuários

### **Conclusão**

Este projeto representa um passo importante na minha evolução como desenvolvedor, focando em escrever código organizado, escalável e funcional com experiência prática em componentização e integração de APIs.

# DIO Clone Project - Study Platform (React.JS Vite - Styled-components)

This project was developed as part of a **Digital Innovation One (DIO)** bootcamp, simulating a study platform with authentication, post feed, and user ranking.

[**(Deploy - Vercel)**](https://clone-dio-reactcli-styled-component.vercel.app/)

### **Applied Architecture**

- **Vite** with **React 19**
- **Styled-components** for styling
- **React Hook Form** + **Yup** for forms
- **Axios** for API consumption
- Component-based organization (not strictly Atomic Design)

### **Quick Start**

```bash
git clone https://github.com/TiagoLeopoldo/nome-do-repositorio.git
cd nome-do-repositorio
yarn install
yarn dev       # Frontend
yarn api       # Backend (JSON-Server)
```

Access: [http://localhost:5173](http://localhost:5173)

### **Why Vite? (Vite vs React CLI)**

The project was initially planned to use **React CLI**, but **Vite** was chosen due to:

- **Superior performance:** much faster initialization and hot reload
- **Productivity:** less waiting time for visual tests and validations (this saved me hours of unnecessary waiting for page rendering)
- **Developer experience:** more agile workflow without compromising React compatibility

### **Tech Stack**

| Category | Technologies         |
| -------- | -------------------- |
| Core     | React 19, Vite       |
| Styling  | Styled-components    |
| Forms    | React Hook Form, Yup |
| HTTP     | Axios                |
| Routing  | React Router DOM     |
| Icons    | React Icons          |
| API      | JSON-Server          |

### **Project Structure**

```
src/
├── components/    # Reusable UI components
├── pages/         # Application pages
├── services/      # API and services
├── styles/        # Global styles
└── ...
```

### **Pages**

- **Home** - Landing page with CTA
- **Login** - User authentication
- **SignUp** - New user registration
- **Feed** - Posts and user ranking

### **Conclusion**

This project represents an important step in my evolution as a developer, focusing on writing organized, scalable, and functional code with practical experience in componentization and API integration.
